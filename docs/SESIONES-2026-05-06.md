# Registro de sesiones (mayo-junio 2026)

Resumen de todo lo construido y aprendido en las sesiones de desarrollo intensivo
del bot CazaSpamBot. Pensado para retomar contexto si se reinicia la conversación.

> Estado al cierre (2026-06-03): **197 tests verde**, bot en producción 24/7,
> 17 detectores, ~41 comandos.

## 1. Detectores añadidos

| Detector | Qué caza | Notas anti-FP |
|---|---|---|
| `obvious_spam_profile` | ≥2 campos de perfil en script no-latín | NFKC + `confusable_homoglyphs` (UTS#39): Cherokee/Math/Thai decorativos NO disparan. Bypass cuenta >1año+foto. 1 solo campo no-latín ya NO banea (era FP) |
| `bio_spam` | Bio con invite t.me/+ + emojis sexuales + keywords (geil/onlyfans/crypto) + idioma extranjero | Score acumulado ≥60. Invite privado = 40 pts. Lee bio vía Telethon `get_participants(search=nombre)` |
| `photos_batch_upload` | ≥3 fotos de perfil subidas en ≤2 min = identidad robada | Bypass si foto más antigua >1 año |
| `commercial_ad` | Anuncio estructurado: multilínea emojis + €/mes + CTA + link | Señales acumuladas ≥60. periodic_money + domestic_offer + urgency |
| `forward_first_msg` | Forward de canal/bot en primer msg o ≤3 min | Ban + reporte |
| `inline_buttons_from_user` | Mensaje con reply_markup (users normales no pueden) | Señal de forward de canal |

## 2. Sistemas de protección anti-falsos-positivos

- **Trust score 0-100** (`db.user_trust_score`): msgs + días + join + warns. ≥70 SKIP total, 40-69 review/degrada, <40 normal. Backfill histórico de los grupos pobló msg_count real (excluyendo service messages).
- **Review-with-buttons**: trust 40-69 + acción severa → DM al admin con ✅Legítimo/❌Spam. Aprende del veredicto.
- **Naive Bayes + Cosine** combinados sobre samples `/spam` y `/legal`.
- **NFKC + confusable_homoglyphs** para nombres decorativos (incidente Cherokee del 29/05 que baneó 100+ legítimos → revertido + blindado).

## 3. Cambios de UX y mensajería

- **Híbrido de quips** (`QUIP_ON_AUTO_BAN=false`): bans automáticos silenciosos, quip solo en bans manuales del admin.
- **Consolidación de ráfagas** (`ban_announce.py`): 3+ bans seguidos → 1 mensaje unificado. TTL 3h.
- **Welcomes graciosos por grupo** (`_WELCOMES_BY_CHAT`): packs temáticos por grupo, frase corta + footer fijo. Castellano correcto (`Bienvenido/a`, nunca `@`).
- **Quips opacos**: no revelan mecanismo de detección (photos_batch, lols_match sin mencionar la fuente).
- **Verificación 3 tiers**: very-legit (skip), normal (reminder 3h + kick +6h), suspicious (kick 30min).

## 4. Comandos nuevos / mejorados

- `/legal` (rename de `/ham`): marca mensaje legítimo, borra el comando, confirma por DM.
- `/ban` y `/unban`: aceptan @username, **text_mention** (users sin username), reply, o id. Fallback Telethon. Borran el welcome huérfano del baneado.
- `/spam`: añade sample + ban + borra welcome.

## 5. Robustez / fixes de fondo

- **`user_signals` resuelve users nuevos** con `get_participants(search=nombre)` (GetParticipantRequest fallaba localmente sin access_hash).
- **Barrido DB de welcomes vencidos** en `cleanup_job` (los `jq.run_once` en memoria se perdían al reiniciar).
- **Reconciliación banned_users↔Telegram** en el job nocturno.
- **Cleanup post-ban solo Bot API** (`forwardMessage` sonda, sin Telethon retrospectivo).
- **Antiflood per-user** graduado por trust (5/8/12 msgs en 10s).
- **Proteger trust del reporter** la cuenta Telethon: whitelist reglas + score≥150 + rate limit 20/h-100/d.

## 6. Seguridad / GitHub

- `.gitignore` protege `.env`, `.env.*`, `*.session`, `*.session-journal`.
- `.env.example` con valores VACÍOS (sin secrets reales).
- Sin secrets hardcoded en `src/`.
- **`TELETHON_ENABLED`** (master switch): false = bot solo con Bot API (sin bio/fotos/reportes/bridge). Para deploys públicos sin la cuenta secundaria. Los detectores Bot-API siguen activos.

## 7. Notificaciones al admin (DM CazaSpamBot)

- Acción antispam: muestra acción tomada (🔨baneado/👢kick/🤐mute/🗑️borrado) + perfil clicable (tg://user, OK en DM privado).
- Borrado manual: quién borró + autor clicable + estado (baneado/sigue) + contenido. Skip si lo borró el propio bot (evita duplicado).
- @admin: reenvía el mensaje reportado al admin DM.

## Reglas de trabajo aprendidas (ver memoria del proyecto)

1. **Nunca acciones masivas sin dry-run** + luz verde por id.
2. **FP > FN**: mejor dejar pasar spam que banear legítimo.
3. **Telethon último recurso** (bio/fotos/reportes/admin_log/histórico). No abusar.
4. **Sin em dashes**, castellano correcto (`Bienvenido/a`).
5. **Sin links clicables a perfiles de spammers** en público (sí en DM admin + top semanal).
6. **Respuestas largas → DM CazaSpamBot al admin**; terminal solo opciones cortas.
